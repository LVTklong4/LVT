# กฎกติกาและหลักปฏิบัติการเขียนโค้ด (Workspace Agent Rules) - LVT

เอกสารนี้กำหนดหลักเกณฑ์ในการทำงานร่วมกันระหว่างผู้ใช้และ AI เอเจนต์ทุกแชทในพื้นที่ทำงานนี้ เพื่อประสิทธิภาพสูงสุดในการประหยัดโทเคน ความแม่นยำในการเขียนโค้ด และการลดผลกระทบระหว่างโมดูล

---

## 1. การประหยัดโทเคนอย่างคุ้มค่า (Token Conservation)
* **แก้ไขโค้ดเฉพาะจุด (Precision Edits)**: ห้ามเขียนทับหรือสร้างไฟล์ใหม่ทั้งไฟล์เมื่อเพียงต้องการแก้ไขจุดเล็กๆ ให้ใช้คำสั่งแก้ไขเฉพาะจุด (`replace_file_content` หรือ `multi_replace_file_content`) โดยเลือกช่วงบรรทัดที่ต้องการสั้นที่สุด
* **ไม่สแกนหรืออ่านไฟล์ฟุ่มเฟือย**: อ่านโค้ดเฉพาะบรรทัดที่เกี่ยวข้องหรือใช้คำสั่งสแกนประเภทไฟล์แบบเจาะจง
* **แยกไฟล์ให้สั้นและกระชับ**: การมีไฟล์ขนาดเล็กหลายไฟล์ (ขนาดต่ำกว่า 500 บรรทัด) ประหยัดโทเคนมากกว่าไฟล์ขนาดใหญ่ไฟล์เดียว (เช่น หน้า 9,000 บรรทัดแบบเดิม) เพราะ AI สามารถเลือกดึงมาอ่านเฉพาะส่วนที่เกี่ยวข้องได้โดยไม่ต้องแบกโค้ดทั้งระบบเข้ามาประมวลผล

---

## 2. การแยกโมดูลเด็ดขาดเพื่อป้องกันผลกระทบย้อนกลับ (Code Decoupling & Safety)
* **ห้ามใช้โค้ดร่วมกันหากมีแนวโน้มทำให้ระบบพังง่าย**:
  * ตัวอย่างเช่น: ส่วนจองรายวัน และ ส่วนจัดการรายเดือน **ต้องแยกไฟล์ Modals และ Layout จากกันอย่างเด็ดขาด** (ห้ามนำมาอ้างอิงจากโค้ดชุดเดียวกันแล้วใช้ Conditional Render ภายใน) เพื่อไม่ให้การแก้ไขฝั่งหนึ่งส่งผลกระทบให้เกิดข้อผิดพลาดกับอีกฝั่ง (Zero Side-Effects)
* **แยกตรรกะออกจาก UI**:
  * โค้ดที่ติดต่อกับ Supabase หรือเก็บ State ต้องอยู่ใน Context (`src/context/...`)
  * โค้ดแสดงผลความสวยงามต้องอยู่ใน Component (`src/components/...`)

---

## 3. การประสานงานและการส่งมอบงานระหว่างแชท (Inter-Chat Handoff)

### 3.1 จะรู้ได้อย่างไรว่าไปแตะโค้ดของแชทอื่น?
* **ตรวจสอบประวัติล่าสุดก่อนเริ่มงาน**: ก่อนที่ AI ในแชทใหม่จะลงมือแก้ไขไฟล์ใดๆ ต้องรันคำสั่งตรวจสอบสถานะของ Git และประวัติการคอมมิตล่าสุดเสมอ (เช่น `git status` และ `git log -n 5`) เพื่อดูว่าแชทอื่นเพิ่งแตะหรือแก้ไฟล์ใดไป
* **ห้ามทำงานชนกันในไฟล์เดียวกัน**: หากพบว่าไฟล์ดังกล่าวกำลังมีงานค้างหรือมีการอัปเดตโครงสร้างโดยแชทอื่น ให้หยุดและแจ้งผู้ใช้ทันที

### 3.2 ขั้นตอนการส่งมอบงานไปแชทอื่น (Handoff Process)
ก่อนที่คุณจะปิดแชทปัจจุบันและเปิดแชทใหม่ ให้สั่งการ AI ตัวเดิมสร้าง **"สรุปการส่งมอบงาน (Handoff Prompt)"** เสมอ โดยใน Prompt จะต้องระบุ:
1. **งานที่ทำสำเร็จแล้ว (What was done)**
2. **โครงสร้างไฟล์ที่เปลี่ยนไป (File updates)**
3. **งานที่ต้องทำต่อทันที (Next Steps)**

*เมื่อคุณเปิดแชทใหม่ ให้ส่ง Handoff Prompt นี้เป็นข้อความแรกสุด AI ตัวใหม่จะสามารถทำงานต่อได้อย่างต่อเนื่องและแม่นยำ 100%*
